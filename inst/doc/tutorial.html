Installation

Installation requires R with a C compiler (Xcode for Mac, RTools for windows).

Currently you need to install from github, in R

library(devtools)
install_github("pwirapati/acdx")

Use install.packages("devtools"), if devtools is not installed.

Alternatively, you can download a version of the R package source from github and install it the usual way.

Data Preparation

The input data are annotated raw counts, with the following components:

  1. Raw counts of the single cell data (not normalized!). It can be filtered (having some genes and cells removed), but there is no requirement to do so.

    The data has to be prepared as sparse matrix format of the Matrix package, with genes as rows and cells as columns. This is the format used by most single-cell analysis pipeline, such as Seurat’s or Alevin’s objects. You have to identify the exact component of the object that is of dgCMatrix class (by running str() on the object). This package is not aware of higher level structures used by various pipelines.

  2. Cell labels to identify groups for aggregation. This is a vector of a factor or strings with the same length and order as the cells (columns) in the raw count matrix.

    There is no requirement that the cell labels have to be done by certain algorithms or in a certain level of details (they can even be random). acdx is a way to evaluate the consequence of a particular choice of cell labeling.

Aggregation and annotations

Assuming rawcounts is the object of raw counts and cellgroup the labels, a cell aggregate object can be computed using:

library(acdx)
ac <- aggr(rawcounts,cellgroup)

See the man page ?aggr for details.

A typical way to prepare the cell group is by combining the sample (library) id with the cell type labels. This will keep the samples separate so that sample-level explanatory variables can be considered.

For example, (assuming cellannot is a cell-level annotation table)

cellgroup <- paste(cellannot$sample_id, cellannot$cell_type,sep="|")

will create strings of aggregate labels by sample identifier and celltypes.

ac is a list of the aggregated data (ac$y) and an annotation data frame (ac$annot). Usually you use more descriptive name for the dataset or cohort.

By default ac$annot contains a column corresponding to the celltype (ac$annot$id) and the number of cells in an aggregate (ac$annot$N).

An additional step that will simplify further analysis is to add columns into the ac$annot data frame. These include the sample id and cell types (which may be part of the cell group definition, but need to be made explicit).

For example,

ac$annot$sample <- gsub("|.*$","", ac$annot$id)
ac$annot$ctype <- gsub("^.*|","", ac$annot$id)

An example aggregate data included in the package can be loaded

data(KUL3)
str(KUL3)

This is from a colorectal cancer dataset available from GSE144735.

There are a number of other columns, such as "patient", "tissue" and "region" (which are all coded in the original single-cell id). There is also a combination column called "ctype_tissue" which groups aggregates from the same cell type and tissue (tumor vs normal) together for the purpose of differential expression analysis.

Model Fitting

To fit the regression model (warning: this performs 99 bootstrap resampling and may take a while, i.e. a couple of minutes on quad-core laptop. Reduce the nboot= value to, say 9, for testing).

fit <- acdx( KUL3$y, X1=KUL3$annot$ctype_tissue, Zf=KUL3$annot$sample,
  G1=KUL3$annot$ctype_tissue, bootid=KUL3$annot$patient, nboot=99,
  verbose=1)

Arguments explanation:

  • KUL3$y is the aggregate data array
  • X1 are the groups to be compared
  • Zf are units of scale factor correction (“normalization”) which in this case is done at the sample (library) level
  • G1 are the groups of dispersion parameters (each group has its own estimate)
  • bootid are units for bootstrapping, in this case the patients. This means that different samples (normal vs tumor core vs border) of the same patient are always selected/discarded together in the bootstrap resampling.
  • nboot number of bootstrap replicates
  • verbose=1 print iteration progression, one line per bootstrap replicate

Differential Expression Queries

The model fitting object only contains estimates of parameters (and their bootstrap replicates).

To rank the genes according to a particular question, we need to specify an “interest functional”. This is a callback function that take the parameter objects and return a score (such as fold change).

For example, we want to find genes that are expressed in tumor epithelial (but not in normal epithelial and other cell types). This is formulated as the log fold change between the tumor epithelial group versus the best of all other groups.

epiT_ifun <- function(beta,...) beta[1,]-apply(beta[-1,],2,max)

This definition is highly specific to how the model is structured. The index 1 in beta correspond to the epithelial tumor group in the beta coefficient matrix (see str(fit) or rownames(fit$beta)).

Arbitrary function can be specified (including the conventional linear contrast).

To perform the ranking:

epiT <- quest( fit, ifun=epiT_ifun )

acplot( KUL3, "SULT2B1",
  xt.key="region",xt.key2="ctype",
  col.key="region",col.map=c(core="red",border="orange3",normal="green3") )

acplot( KUL3, "CLDN2",
  xt.key="region",xt.key2="ctype",
  col.key="region",col.map=c(core="red",border="orange3",normal="green3") )
