acdx: Quick Tutorial

This is an example that illustrate a typical workflow in acdx using a publicly available dataset GSE144735 (Lee et al. 2020).

All codes are to be run inside R.

Installation

One-time setup:

library(devtools)
install_github("pwirapati/acdx")

This requires R with a C compiler (Xcode for Mac, RTools for windows).

Check by loading the package:

library(acdx)

Data preparation

Download the raw counts and annotations of GSE144735 from GEO into the current R working directory. These two files will take up ~55Mb of file space and bandwidth.

  ## downloads large files from GEO
  baseurl <- "ftp://ftp.ncbi.nlm.nih.gov/geo/series/"
  gsesuppl <- "GSE144nnn/GSE144735/suppl/"
  download.file(paste0(baseurl,gsesuppl,
    "GSE144735_processed_KUL3_CRC_10X_annotation.txt.gz"),
    "KUL3_annotation.txt.gz")
  download.file(paste0(baseurl,gsesuppl,
    "GSE144735_processed_KUL3_CRC_10X_raw_UMI_count_matrix.txt.gz"),
    "KUL3_rawUMI.txt.gz")

For brevity, we will refer to this dataset as KUL3 (KU Leuven 3â€™ protocol) to distinguish it from similar dataset in Lee et al. (2020). Various R objects derived from this dataset will have the stem "KUL3" in their names. (This is just a convention for convenience.)

Check if the files are in current working directory:

list.files()

Cell aggregation

Load the package (if not already):

library(acdx)

Read the cell annotation data:

a <- read.delim("KUL3_annotation.txt.gz")

Examine the content briefly:

str(a)
head(a)

The aggregates are defined by a combination of sample and cell types identifiers.

In this example, we will use the columns Sample and Cell_type.

There is no particular reason for this choice aside from having small, reasonable example. You can use the Cell_subtype columns (with more categories), or new cell type labels from your own analysis.

Perform the aggregation (can take few minutes):

KUL3 <- greg("KUL3_rawUMI.txt.gz",a$Sample, a$Cell_type )

On completion you should see "processed: 33694 genes".

Notes:

  1. If the raw counts has been loaded as a sparse matrix object (dgCMatrix format commonly used by various single-cell packages) or even as dense R matrix, you can replace the file name above with the object name. acdx is not aware of higher-level structure used by various other pipelines, so you have to locate the object part that has raw counts in dgCMatrix format, such as using str().
  2. Make sure the barcode IDs of the sample and cell type annotation vectors are in the same order as those of the raw counts matrix. In this example, the first line (column header) of the tab-delimited raw counts matches the row names of the cell annotation file.

Examine the content of the aggregate object:

str(KUL3)

It has two matrices, KUL3$N (small) and KUL3$y (large).

You can print the small one:

KUL3$N

which contains the cell counts of every sample and cell type combination.

The much larger KUL3$y object is a four-dimensional array of (mean,SE2) x samples x genes x celltypes. To take a peek of the first two entries of each dimension:

KUL3$y[,1:2,1:2,1:2]

u and v are mean and SE2 of each sample-gene-celltype combination.

A more interesting example is the mean raw counts of highly expressed gene:

KUL3$y[1,,"MALAT1",]

It is more convenient to see them visually:

acgplot(KUL3,"MALAT1")

Sample Annotation Preparation

There are other information about the samples that we would like to use. In this example, it is the column Class of the annotation table which contains information about the tissue type:

table(a$Sample,a$Class)

They need to be reduced to a table with sample-level information (with the same sample order as used in the aggregate).

We can add them to the KUL3 aggregate object.

KUL3$a <- data.frame(row.names=rownames(KUL3$N))
KUL3$a <- a[match(rownames(KUL3$a),a$Sample),"Class"]

KUL3$a

Suppose we donâ€™t want to distinguish Border and Tumor. We can derive another variable.

KUL3$a$tissue <- sub("Border","Tumor",KUL3$a$Class)
KUL3$a$Class <- sub("Tumor","Core",KUL3$a$Class) # rename the original
names(KUL3$a)[1] <- "region"          # change the column name

KUL3$a

These can be used to annotate the plot:

acgplot(KUL3,sample_tags=KUL3$a$region, "EPCAM")
acgplot(KUL3,sample_tags=KUL3$a[,c("region","tissue")], "EPCAM")

mycolor <- c(Normal="green4",Border="orange",Tumor="red",Core="red2")
acgplot(KUL3,sample_tags=KUL3$a$tissue,
  col.dot=mycolor[KUL3$a$tissue], pch=20,cex=2,"EPCAM")

Fit regression models

First we need to define a design matrix. There are many such matrices depending on the questions, and choice of parameterization (dummy coding). This can be a complex subject with various tools (similar to other differential expression or regression software).

For a quick example, we will use one with two columns corresponding to Normal and Tumor.

KUL3$X1 <- cbind( Normal=ifelse(KUL3$a$tissue=="Normal",1,0),
                  Tumor=ifelse(KUL3$a$tissue=="Tumor",1,0))

KUL3$X1

Itâ€™s important to name the columns to allow reference to them later.

We can store the matrix anywhere, but itâ€™s convenient to keep it inside the KUL3 object. The number 1 after X is just to keep track in case we have many other design matrices. More descriptive names can be used.

Note: This parameterization has no intercept, but it is convenient because the parameters correspond to average expression within the groups. The same fit can be achieved with an intercept (all 1â€™s) and a contrast (e.g., Tumor is 1, Normal is -1). The choice depends on what question we want to address later.

Now we fit X1 to the data, separately for each gene in each cell type, and repeat them in 100 bootstraps (including the original sampling as the first instance).

This will take some time to run (92 seconds in a laptop with 4-core CPU i7-6700HQ) and will produce an R object with ~800Mb of size. Test first with smaller value of n_boot (the default is one). verbose=1 print a hash mark for every bootstrap, to get an idea about the progression.

system.time( KUL3$fit1 <- acdx( KUL3, KUL3$X1, n_boot=100, verbose=1) )

object.size(KUL3$fit1)

The size is proportional to the number of bootstraps. We keep them in memory because they are re-usable for answering different questions.

The pattern can be more complex. For example, if we want genes that are high in tumor of both epithelial and stroma, low elsewhere, but indifferent about whether it should be low or high in normal stroma:

top(
  questmm(KUL3$fit1$beta,
  H=c("Tum.*,Epi","Tum.*,Stroma"),N=c("Nor.*,Str")))

Notes:

  1. The pattern are specified as regular expressions to H= (high) , L= (low) and N= (neutral). The default is L, but can be change by default="H" or default="N".The ranking of the genes are based on min(coef_in_H)-max(coefs_in_L). Coefficients in N are ignored.

  2. We can assign the output of questmm to a named object of repeated invocation of top is needed. A useful option to top (in addition to n= for the number top ranking genes) is sort.by=.

To get the gene names of those with FWER â‰¤ 0.05 (this is a multiple-testing corrected bootstrap p-value):

mygenes <- rownames(
  subset(
    top( questmm(KUL3$fit1$beta, 
      H=c("Tum.*,Epi","Tum.*,Stroma"),N=c("Nor.*,Str"))),
    FWER <= 0.05))

mygenes

pdf(file="myplot.pdf",width=8,height=12,pointsize=8)
par(mfrow=c(6,1),cex.axis=1.2,cex.lab=1.2,mar=c(8,5,3,3))
